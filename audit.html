Audit Trail & Compliance Logging

Recordkeeping requirements, audit log standards, retention schedules, access controls, and independent audit procedures.
Regulatory Basis
BSA recordkeeping requirements under 31 CFR 1010.430 require that all required records be maintained in a manner that makes them accessible and retrievable within a reasonable time (generally defined as 72 hours for regulatory examination purposes). Destruction of records subject to a legal hold is a federal obstruction offense.

Audit Log Standards

All compliance system audit logs must meet the following minimum technical standards:

Required Log Fields

  • Timestamp — UTC timestamp with millisecond precision
  • User ID — Authenticated user or service account identifier
  • Action — Specific action performed (e.g., VIEW, EDIT, DELETE, APPROVE, FILE, OVERRIDE)
  • Object Type — Entity type acted upon (e.g., Customer, Alert, SAR, CTR, Sanctions Hit)
  • Object ID — Unique identifier of the object affected
  • Previous Value — Where applicable, the value before the change
  • New Value — Where applicable, the value after the change
  • IP Address / Session ID — Origin of the action for security correlation
  • Result — Success or failure status of the action
Log Integrity
Compliance audit logs must be write-once / append-only. No user — including system administrators — should have the ability to modify or delete audit log entries. Logs must be stored in a separate, access-controlled audit log repository with integrity verification (hash chaining or equivalent).

Log Monitoring & Alerting

The following events must trigger automated alerts to the Compliance Operations team:

  • Unauthorized access attempt to SAR or blocked case records
  • Bulk export of compliance records outside of approved workflows
  • Any attempt to delete or truncate audit log files
  • Change to TMS monitoring rule parameters (thresholds, suppressions)
  • System user privilege escalation in compliance systems
  • Screening system connectivity failure exceeding 5 minutes

Independent Compliance Audit

The BSA/AML program must be subject to periodic independent testing to assess its adequacy and effectiveness. "Independent" means the function must be organizationally separated from the compliance program it is reviewing.

Exam-Ready Documentation Standard

The institution must maintain a BSA/AML Examination Package — a ready-to-produce set of documents that can be delivered to regulators within 5 business days of an examination commencement notice. The package must include:

  • Current BSA/AML Policy and Procedures (board-approved)
  • BSA Officer designation and organizational chart
  • Last two years of audit reports and management responses
  • Training completion records for the current and prior year
  • Written Enterprise Risk Assessment (current)
  • TMS rule inventory and change log
  • SAR filing statistics for the examination period
  • CTR filing statistics and exemption list
  • OFAC screening system documentation and validation testing results
  • Beneficial ownership certification completion rates
Legal Hold
Upon receipt of a subpoena, legal hold notice, regulatory examination notice, or internal escalation involving potential litigation, all destruction of relevant records must be immediately suspended. The Legal department must be notified within 24 hours of any hold-triggering event. Preservation obligations supersede normal retention schedules.

Examination Cooperation
Financial institution personnel must cooperate fully with regulatory examinations. Examiners have the right to access all BSA records without advance notice during an on-site examination. Designated examination liaisons should coordinate all document requests and examiner interviews through the BSA Officer team.
